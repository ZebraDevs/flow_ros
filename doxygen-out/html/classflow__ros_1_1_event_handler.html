Synchronized input container type.

std::tuple<...> with data from each synchronized input. Inputs are ordered with respect to the ordering of the capture buffers from which they were sources, specified by SubscriberTuple

◆ Output

template<typename PublisherTuple , typename SubscriberTuple , template< typename > class OutputContainerTypeInfoTmpl = DefaultOutputContainerTypeInfo>
using flow_ros::EventHandler< PublisherTuple, SubscriberTuple, OutputContainerTypeInfoTmpl >::Output = typename detail::EventHandlerOutputType<PublisherTuple>::type

Output data type.

Message type returned from event callback. If:

  • PublisherTuple specifies a single output, then this is a OutputMsgT::Ptr associated with that publisher

PublisherTuple specifies multiple outputs, then this is a std::tuple<Output0MsgT::Ptr, ... , OutputNMsgT::Ptr> associated with those publishers in the order specified by PublisherTuple.

Event updater method.

On each call, this method attempts input synchronization. The behavior that follows depends on user callbacks setup on construction. If synchronization succeeds, the main event (execution) callback is invoked.
Subscribers which wait for data may wait until timeout unless synchronization becomre possible
